# SECURITY.md

## 支持的版本 (Supported Versions)

仅对项目的 **最新稳定版** 提供安全更新支持，旧版本不再提供主动安全补丁。强烈建议用户及时更新至最新版本，以规避潜在安全风险。

## 漏洞披露流程 (Vulnerability Disclosure Process)

### 1. 如何报告漏洞 (How to Report a Vulnerability)

如发现潜在安全漏洞（包括但不限于代码漏洞、数据泄露风险、恶意代码注入等），请立即通过 **私密、安全的渠道**报告。切勿**公开**披露漏洞（如在 Issues、公开聊天群、社交媒体等），以免被恶意利用。

#### 推荐报告渠道 (Recommended Reporting Channels)

- GitHub Issues：通过仓库的「Security」标签提交 **机密安全咨询** [MaaAutoNaruto 机密安全咨询](https://github.com/duorua/narutomobile/security/advisories)

- 项目聊天群：在官方开发群（QQ：984076079）中私信项目维护者，主题注明「安全漏洞报告」。

#### 漏洞报告需包含的信息 (Required Information for Vulnerability Reports)

为帮助我们高效复现并修复漏洞，请提供以下详细信息：

1. 漏洞详细描述（如类型、影响范围、潜在危害）。

2. 分步复现步骤（包括环境配置、操作流程等）。

3. 与漏洞相关的截图、视频或日志文件。

4. MaaDebugger和VSCODE扩展Maa Pipeline运行情况。

5. 已知的缓解措施或临时解决方案（如有）。

## 数据安全说明 (Data Security Statement)

### 1. 数据收集原则 (Data Collection Principles)

`MaaAutoNaruto` 项目 **不主动收集、存储或传输任何用户敏感数据**，包括但不限于游戏账号密码、个人身份信息、设备信息、游戏进度数据等。工具的所有操作均在用户本地设备完成。

### 2. 用户数据保护建议 (User Data Protection Recommendations)

- 切勿在与本项目相关的任何第三方插件或辅助工具中输入游戏账号密码等敏感信息。

- 保持设备操作系统、模拟器及 `MaaAutoNaruto` 工具为最新版本，规避系统级漏洞。

- 不与不可信人员共享设备、模拟器账号或工具配置文件。

- 通过合法官方渠道下载工具（仅从项目 GitHub Release 页面），避免恶意修改版本。

## 使用安全提示 (Usage Security Notes)

### 1. 游戏合规风险 (Game Compliance Risks)

本工具基于图像识别开发，不修改游戏核心文件或内存数据，但使用自动化工具可能违反《火影忍者手游》用户协议。项目团队对使用工具导致的任何后果（如账号封禁、限制登录等）不承担责任，用户需自行承担使用风险。

### 2. 设备安全 (Device Security)

- 避免在带有恶意软件（如病毒、木马）的设备上运行工具，防止工具被劫持或篡改。

- 除非必要，不向工具授予不必要的系统权限（如 root 权限、管理员权限）。

- 使用完毕后及时关闭工具，避免长期后台运行引发潜在安全风险。

## 依赖项安全 (Dependency Security)

项目依赖 MaaFramework、MFAAvalonia 等开源框架及库，项目团队将：

1. 定期监控依赖项目的安全公告，发现漏洞时及时更新至安全版本。

2. 在 CI/CD 流程中加入依赖安全检测，拦截存在漏洞的依赖版本。

3. 在 `requirements.txt`、`package.json` 等文件中明确列出所有依赖，保证透明度。

## 贡献者安全规范 (Contributor Security Guidelines)

贡献者（包括代码提交者、文档编辑者等）必须遵守以下安全要求：

1. 不得向项目中植入恶意代码（如后门、数据窃取模块）。

2. 遵循安全编码规范，避免常见漏洞（如缓冲区溢出、注入攻击）。

3. 提交代码前自行检测安全风险，必要时附上相关测试报告。

4. 不将敏感信息（如个人联系方式、私钥）写入代码或文档。

5. 开发过程中发现安全问题，及时向项目维护者报告。
